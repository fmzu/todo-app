# TODO作成/一覧 仕様テンプレート

- 目的: タスクを登録・一覧・状態管理する理由を一文で
- 前提/制約: 認証有無、永続化手段（例: ローカルストレージのみ）、対応ブラウザなど
- 入力: タイトル(必須)、メモ(任意)、期限(任意)、完了フラグ(チェックボックスで切替)などフィールドと制約
- 出力: 登録済みタスクの配列構造、成功/失敗メッセージの形式
- 状態変化: `todos`状態の追加/更新/削除、フィルタ状態の保持など
- UIメモ: フォーム＋リスト＋フィルタ（All/Active/Done）、完了トグルはチェックボックス、空時プレースホルダ、ソート想定
- エラー/バリデーション: タイトル必須、文字数上限、期限の過去日警告など
- 未決事項: 削除確認の有無、完了タスクの表示期間、ソート基準など

## 追加メモ
- 実装開始後に決まった仕様変更は箇条書きで追記
- AIに投げたプロンプトや回答で得た決定事項があれば引用で貼る
