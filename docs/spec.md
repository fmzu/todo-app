# 集団タスク閲覧アプリ 仕様書

## 概要
- 目的: チーム全員の「今日やるタスク」を一画面で俯瞰し、担当・期日・状態を素早く把握する
- レイアウト: Googleカレンダー風。横にメンバー列、各列にそのメンバーのタスクカードを追加順で縦並び（時間軸なし）。メンバーが多い場合は横スクロール
- スコープ: 自分のタスクは編集可。他メンバーのタスクは閲覧のみ（初期リリース）
- 対象: 小〜中規模チームのメンバーとリーダー。日単位の朝会/終業前チェックで使用

## 主要機能
- メンバー列表示: メンバー名と未完数などのサマリをヘッダーに表示（他メンバー列は閲覧専用）
- タスク表示: タイトル、備考（任意）、期日（日付のみ）、完了チェック
- タスク追加/編集/削除: メンバー列ごとに行う
- 完了/未完の切り替え（チェックボックス）
- 権限: 自分のメンバー列のみ追加・編集・削除が可能。他メンバー列は閲覧のみ
- 並び: 追加順のみ（手動並び替えなし）
- 期日: 日付を保持（時間帯は扱わない）。入力は当日を初期値にし、当日以降の日付を選択可能

## 非機能・表示要件
- レスポンシブ: PCは横スクロールで複数メンバーを並列、モバイルはメンバーを縦スタック＋左右スワイプ検討
- 視認性: 枠線は控えめ、カード背景・ホバーで区切りを示す
- 入力: フォーム枠は最小限（必要ならフォーカス時のみ表示）
- 言語: 日本語表示
- パフォーマンス: 数十人×数十タスク程度をスムーズに表示できる前提

## データモデル案
- Member: `{ id: string; name: string; }`
- Task: `{ id: string; memberId: string; title: string; note?: string; dueDate: string (YYYY-MM-DD); done: boolean; createdAt: string; }`

## 操作フロー例
- タスク追加: 列ヘッダーまたは列末尾の「＋」からタイトル必須、備考/期日オプション（期日は当日を初期値に当日以降を選択）
- タスク編集: 自分の列のカードのみ編集可（クリックまたはメニューから編集）
- 他メンバーのタスクは閲覧のみ
- 完了チェック: カード左のチェックで即反映
