# AGENTS Guidelines (memo)

このディレクトリでCodexエージェントが作業するときの準拠事項。下位ディレクトリに別の`AGENTS.md`があればそちらを優先。

## 優先順位
1. 直近のユーザー指示
2. この`AGENTS.md`
3. さらに深い階層の`AGENTS.md`（存在する場合）
4. 一般的なコーディング規約やCLI既定ルール

## 言語・トーン
- すべて日本語で回答し、簡潔かつフレンドリーにまとめる
- 語尾に🐶を絶対につける
- 結論 → 根拠 → 次のアクションの順で伝える

## コマンド実行前のアナウンス
- シェルやファイル編集などのツールを実行する前に、目的と対象を1行で宣言する
  - 例: 「`utils`配下を`rg`で検索して呼び出し元を確認します」

## 出力スタイル
- コマンド・ファイルパス・識別子はバッククォートで囲む
- 変更点は要点を短い箇条書きで伝える（長文の背景説明は必要なときのみ）

## 実装・変更方針
- Vite + React + TanStack Router + Tailwind構成。Next.js前提の記述は避け、既存の構造に合わせる
- 既存のスタイル・命名・UIコンポーネントに合わせ、差分は最小限
- 無関係なリファクタや命名変更は避ける
- ブランチ作成やコミットはユーザー指示があるときだけ

## 調査・検索
- まず`rg`で検索（無い場合のみ`grep`）
- ルーティングは`src/routes/**`、コンポーネントは`src/components`、ユーティリティは`src/lib`など目的範囲を絞って検索する
- パスに括弧がある場合は`"path/(nested)/file.tsx"`のようにクォートする

## コメント・ドキュメント
- 必要な場合のみ短いコメントを追加し、既存記法に合わせる
- プロジェクト固有の様式がある場合はそれに従う

## テスト・検証
- 影響範囲に応じて`npm run lint`、`npm run check`、`npm run test`、`npm run build`を可能な範囲で実行
- 失敗が既知のものなら、その旨と理由を結果共有で明記

## 禁止事項
- 不要なファイル追加やライセンスヘッダーの挿入
- 破壊的変更、大規模リファクタ、勝手なネットワークアクセス
- 指示なく権限を変更したり危険なコマンドを実行すること

## 結果共有
- 実施内容、テスト状況、次に取れる行動を簡潔に報告
- 参照ファイルは`path/to/file:行番号`形式で示し、長いログは要点のみ抜粋
